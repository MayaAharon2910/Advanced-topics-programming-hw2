BONUS FEATURES - ASSIGNMENT 2
=============================

This file lists the additional features implemented beyond the minimum
Assignment 2 skeleton and explains how to verify them.

1. Visual Simulation (External Python Utility)
----------------------------------------------

Feature summary

A standalone Python 3 script `visualize_map.py` is provided in the project
root. It loads any 3-D `.npy` map file produced by (or used as input to)
drone_mapper_simulation and renders an interactive 3-D voxel plot using
Matplotlib. Each voxel is coloured by its occupancy state:

  -3  PotentiallyOccupied  →  orange
  -2  OutOfBounds          →  gray
  -1  Unmapped             →  light blue
   0  Empty                →  white smoke
   1  Occupied             →  dark red

The script is completely standalone — it is NOT required by the core
simulation build and does not affect CMake, vcpkg, or any C++ target.

Requirements

    pip install numpy matplotlib

How to verify

After building the project, run the simulator once to generate output maps:

    ./build/drone_mapper_simulation

Then launch the visualiser on an input map:

    python3 visualize_map.py data_maps/single_voxel_x2_y4_z2.npy

Or on a generated output map:

    python3 visualize_map.py output_results/output_map_0.npy

To show all non-negative voxels (Empty + Occupied + PotentiallyOccupied):

    python3 visualize_map.py output_results/output_map_0.npy --threshold 0

An interactive 3-D window will open. You can rotate the voxel grid by
clicking and dragging, and zoom with the scroll wheel.

Relevant file

    visualize_map.py  (project root)

Note: this bonus was NOT submitted in Assignment 1.


2. Cross-resolution map comparison support
------------------------------------------

Feature summary

`MapsComparison::compare()` compares maps in world-space coordinates and
uses each map's `MapConfig` (boundaries, offset, resolution). When the
original and target maps use different resolutions, the comparison picks a
sampling step aligned with both maps by taking the GCD of their integer-cm
resolutions, falling back to the finer resolution when needed.

Relevant files

    src/MapsComparison.cpp
    include/drone_mapper/MapsComparison.h
    tests/components/MapsComparison_test.cpp

How to verify

    cmake --build build
    ./build/drone_mapper_simulation_test --gtest_filter=MapsComparison.CrossResolutionBonus


3. Full Assignment 2 output layout
----------------------------------

The simulator writes the complete required output structure:

    <output_path>/simulation_output.yaml
    <output_path>/output_results/
        output_map_<n>.npy
        error_log.txt

How to verify

    cmake --build build
    ./build/drone_mapper_simulation
    cat simulation_output.yaml
    ls output_results/


4. Synchronous error logging
----------------------------

Errors are written immediately (no buffering) to:

    output_results/error_log.txt

How to verify

    ./build/drone_mapper_simulation_test --gtest_filter=Integration.*Failure*


5. Ported mapping pipeline from HW1 (Hybrid Exploration: Sweep + BFS)
----------------------------------------------------------------------

The Assignment 1 Adaptive Sweep + BFS Frontier algorithm was ported in full
into MappingAlgorithmImpl, wired through all Assignment 2 interfaces.

How to verify

    ./build/drone_mapper_simulation_test --gtest_filter=Integration.FullFlowRealMapping
